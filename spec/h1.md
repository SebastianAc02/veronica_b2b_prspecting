# Spec H1 — Bootstrap infra

## Goal

Al terminar H1 existe una app Next.js con basic auth corriendo en `localhost:4500`, conectada a un repositorio Git, y deployada en Fly.io con una URL pública accesible.

## Context

El repo de destino es `https://github.com/SebastianAc02/veronica_b2b_prspecting.git`. El repo está vacío — todo se crea en este H. Stack fijo definido en `CLAUDE.md`, no se renegocia. Naming conventions del `CLAUDE.md` aplican desde el primer commit.

## Constraints

1. No hay schema de DB todavía — eso es H2.
2. No hay workers reales de Inngest — solo el endpoint `/api/inngest` wired, nada más.
3. No hay features de producto — ni engines, ni UI de jobs, ni CSV.
4. Basic auth solo con password en env var — no auth completa, no OAuth.
5. El Dockerfile es solo para la app Next.js — el crawler Python es servicio separado, no va en H1.
6. No hay setup de R2 todavía.
7. Deploy a Fly.io es manual — sin CI/CD automático en H1.

## Tasks

1. Crear el proyecto Next.js 14 con TypeScript, Tailwind y Shadcn/ui. Conectar al repo `https://github.com/SebastianAc02/veronica_b2b_prspecting.git`. Primer commit desde el inicio.
2. Crear DB en Neon (free tier). Instalar y configurar Drizzle ORM. Connection string en `.env.local`.
3. Crear cuenta Inngest. Instalar SDK. Crear cliente y endpoint `/api/inngest`.
4. Crear app en Fly.io. Escribir `fly.toml` y Dockerfile para Node. Primer `fly deploy` exitoso.
5. Implementar basic auth middleware — bloquea todas las rutas excepto `/api/inngest`.

## Done when

1. `pnpm dev` levanta la app en `localhost:4500` sin errores.
2. El browser pide password al entrar — basic auth funciona.
3. La base de datos existe en Neon y el proyecto se conecta sin error.
4. La app está deployada en Fly.io y la URL pública responde con basic auth.

## Verification

1. Correr `pnpm dev`, abrir `localhost:4500` — browser pide password, con password correcto entra y muestra algo.
2. Correr un script de health que haga una query simple a Neon y loguee "OK" en consola.
3. Abrir la URL pública de Fly.io — misma experiencia que local.
